# 패유기 개선 사항 리스트

남은 개선 과제만 정리. 완료 항목(오디오 시스템·AI 난이도·역 판정 정합성[순/초일색 보라 제외 등]·페이싱 타이머·입력 키 통일[Z/Space·ESC, ENTER/X 고아키 제거]·모바일 게임패드[터치 조작감 개선 갈음]·대규모 리팩토링 등)은 git 히스토리 참조.

## 1. 드로우 로직 보정 (캐릭터별 운/개성 반영)
- **현재:** `generateDeck`에서 전체 패를 섞고 순서대로 뽑는 완전 무작위.
- **개선:** 캐릭터의 `luck` 스탯에 따라 덱 상위 N장을 몰래 살펴 현재 손패와 비교, 가장 가치 있는(유효패) 패를 골라 드로우. 나머지는 덱에 되돌려 전체 구성비를 훼손하지 않음.
- **기대:** 캐릭터 설명에 맞는 '패를 뽑는 운'을 시각적·체감상 구현.
- **참고:** 본 항목(캐릭터별 운)은 미구현. 단, **난이도 쪽 절반은 구현됨** — 쉬움 전용 플레이어 드로우 어시스트(`BattleEngine.drawTiles`의 peek-재배열 + `YakuLogic.rateTileForHand` 높은 패 지향 평가, `BattleConfig.RULES.DRAW_ASSIST`). 캐릭터 `luck` 스탯은 같은 메커니즘에 발동 확률만 더하면 됨.

## 2. 상황 인지형 캐릭터 대사
- **현재:** 타패 대사가 상황(필요한 패를 버리는지 등)과 무관하게 랜덤 출력. (과거 상황 인지 로직은 성능 저하로 기각.)
- **개선:** 무거운 샹텐 계산 대신 가벼운 휴리스틱으로 패 가치를 평가해 대사 분기(예: 버린 패 ±2 범위에 손패가 있는가 — O(N) 1회). 드로우/타패 시점에 1번만 계산.
- **참고:** `YakuLogic.checkTenpai`(순수 함수)와 텐파이 인식형 AI가 도입돼 기반은 마련됨. 대사 분기는 CPU 턴당 1회만 도는 경로라 비용 부담 적음.

## 3. 난이도 게임 중 변경 시 처리 (미정)
- **현재:** 툴바 난이도 토글은 `Game.saveData.difficulty`만 갱신. 실제 적용은 전투 시작(`BattleScene.init`)에 1회 — 진행 중인 매치엔 반영 안 되고 **다음 전투/컨티뉴부터** 적용됨.
- **문제:** 전투 중 토글한 플레이어는 즉시 효과를 기대하나 이번 판엔 변화 없음 → "버튼 고장?" 혼란 가능.
- **방향(미결정):** 전투 중에는 토글 **버튼 비활성화**(현재 처리 불가 상태를 상태로 표현). 사용자 선호. 트레이드오프: "다음 판 난이도 미리 변경"이 막힘 → "전투 중 비활성 + 전투 종료 시 자동 활성"이 무난. (대안: 즉시 재계산 — 싱글플레이라 악용 문제 없음.)
- **구현:** `Game.changeScene`에서 현재 씬이 `BattleScene`이면 `#difficulty-btn` `disabled` 토글 + 비활성 스타일.

## 4. 히든 보스(마유) 난입 연출 — 원본 정합 (차후)
- **현재:** 난입 시 `EncounterScene` `mode:'CHALLENGER'` → 정체불명 실루엣(`face/MAYU_unknown.png`)을 ~3초(`textTimer > 180`) 또는 입력까지 표시(`drawChallenger`, state `-1`) → **조우 대사 없이 바로 `BattleScene`**. 진입점은 `endingScene.js:checkTrueEnding`(엔딩 일러스트 후, 노멀 크레딧이 나올 자리에서 크레딧 대신 난입).
- **문제:** 원본 게임의 히든 보스 등장 연출과 다름 → 원본에 맞춰 수정 필요. (구체적 차이는 원본 확인 후 정리 예정.)
- **타이밍은 유지:** "엔딩 일러스트 → 크레딧 대신 난입 → 즉시 전투 → 격파 후 정체 공개 대사(`TRUE_ENDING_CLEAR`)" 흐름 자체는 의도대로 OK. 바꿀 건 **난입 화면의 연출(비주얼/사운드/연출 길이 등)**.
- **위치:** `encounterScene.js`(state `-1` 분기, `drawChallenger`, `ChallengerConfig`), `endingScene.js:checkTrueEnding`.
- **참고:** CHALLENGER 모드 `init`에서 대사(`DialogueData[key]`)를 로드하지만 state `-1`이 바로 전투로 보내 **사용 안 함**(불필요 로드) — 연출 수정 시 함께 정리 가능.
