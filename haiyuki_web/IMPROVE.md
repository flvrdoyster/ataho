# 패유기 개선 사항 리스트

남은 개선 과제만 정리. 완료되어 제외된 항목은 git 히스토리 참조.

## 1. 드로우 로직 보정 (캐릭터별 운/개성 반영)
- **현재:** `generateDeck`에서 전체 패를 섞고 순서대로 뽑는 완전 무작위.
- **개선:** 캐릭터의 `luck` 스탯에 따라 덱 상위 N장을 몰래 살펴 현재 손패와 비교, 가장 가치 있는(유효패) 패를 골라 드로우. 나머지는 덱에 되돌려 전체 구성비를 훼손하지 않음.
- **기대:** 캐릭터 설명에 맞는 '패를 뽑는 운'을 시각적·체감상 구현.
- **참고:** 본 항목(캐릭터별 운)은 미구현. 단, **난이도 쪽 절반은 구현됨** — 쉬움 전용 플레이어 드로우 어시스트(`BattleEngine.drawTiles`의 peek-재배열 + `YakuLogic.rateTileForHand` 높은 패 지향 평가, `BattleConfig.RULES.DRAW_ASSIST`). 캐릭터 `luck` 스탯은 같은 메커니즘에 발동 확률만 더하면 됨.

## 2. FX 스타일 디테일업 (원본 게임 정합)
- **목표:** 전투 FX(스킬/타격/리치 등)를 원본 게임에 더 가깝게 다듬기.
- **현재:** `BattleEngine.playFX('fx/...')` + `SkillFlows`에서 슬래시/힐/리치 등 FX 재생. 스프라이트·타이밍·스케일이 원본과 일부 차이.
- **참고:** 구체적 차이는 원본 캡처 확보 후 정리. 위치 후보 — `js/views/battleRenderer.js`(FX 드로우), `js/logic/skillRegistry.js`(SkillFlows), `assets/fx/*`.

## 3. 스탭롤 완전 구현
- **목표:** 엔딩 후 스태프롤(크레딧)을 원본 수준으로 완성.
- **현재:** `CreditsScene` 존재(NORMAL/TRUE 분기). 스크롤 연출·내용이 미완 추정.
- **참고:** 원본 크레딧 진행/표기 확인 후 항목 상세화.

## 4. 배틀 결과(RESULT) 창 개선
- **목표:** 승/패 결과 화면(역·점수·데미지 표기)의 레이아웃·연출 개선.
- **현재:** 승리 시퀀스 결과 표시는 `BattleSequencer.startWinSequence`(REVEAL_HAND/REVEAL_URA 등 스텝)로 구성.
- **참고:** 개선 범위(가독성/원본 정합/정보량) 결정 필요.

## 5. 별도 튜토리얼 추가 검토
- **목표:** 규칙(패·역·스킬·조작)을 가르치는 별도 튜토리얼 도입 여부 검토.
- **참고:** 도입 시 진입점(타이틀 메뉴?)·범위(인터랙티브 vs 정적 안내) 결정 필요. 우선 "검토" 단계.
