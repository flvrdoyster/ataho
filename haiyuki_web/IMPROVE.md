# 패유기 개선 사항 리스트

남은 개선 과제만 정리. 완료 항목(오디오 시스템, AI 난이도, 대규모 리팩토링 등)은 git 히스토리 참조.

## 1. 드로우 로직 보정 (캐릭터별 운/개성 반영)
- **현재:** `generateDeck`에서 전체 패를 섞고 순서대로 뽑는 완전 무작위.
- **개선:** 캐릭터의 `luck` 스탯에 따라 덱 상위 N장을 몰래 살펴 현재 손패와 비교, 가장 가치 있는(유효패) 패를 골라 드로우. 나머지는 덱에 되돌려 전체 구성비를 훼손하지 않음.
- **기대:** 캐릭터 설명에 맞는 '패를 뽑는 운'을 시각적·체감상 구현.
- **참고:** 본 항목(캐릭터별 운)은 미구현. 단, **난이도 쪽 절반은 구현됨** — 쉬움 전용 플레이어 드로우 어시스트(`BattleEngine.drawTiles`의 peek-재배열 + `YakuLogic.rateTileForHand` 높은 패 지향 평가, `BattleConfig.RULES.DRAW_ASSIST`). 캐릭터 `luck` 스탯은 같은 메커니즘에 발동 확률만 더하면 됨.

## 2. 상황 인지형 캐릭터 대사
- **현재:** 타패 대사가 상황(필요한 패를 버리는지 등)과 무관하게 랜덤 출력. (과거 상황 인지 로직은 성능 저하로 기각.)
- **개선:** 무거운 샹텐 계산 대신 가벼운 휴리스틱으로 패 가치를 평가해 대사 분기(예: 버린 패 ±2 범위에 손패가 있는가 — O(N) 1회). 드로우/타패 시점에 1번만 계산.
- **참고:** `YakuLogic.checkTenpai`(순수 함수)와 텐파이 인식형 AI가 도입돼 기반은 마련됨. 대사 분기는 CPU 턴당 1회만 도는 경로라 비용 부담 적음.

## 3. 모바일 환경 터치 조작감 개선 (Mobile UX/UI)
- **현재:** PC 마우스 조작 우선 설계라 터치 시 조작이 애매하거나 오터치 발생(패 선택 딜레이, 좁은 터치 영역 등).
- **개선 방향:**
  - **터치 영역(Hitbox) 확장:** 패·버튼의 클릭 판정 영역을 시각 크기보다 조금 크게 잡아 오터치 방지.
  - **상태 피드백 강화:** 터치 순간 즉각적인 시각 피드백(확대·색 변화·햅틱 진동 API)으로 조작 확신 부여.
  - **UI 레이아웃 조정 (선택):** 모바일 화면 비율에 맞춰 손가락이 닿기 편한 위치로 메뉴·버튼 반응형 미세 조정.
- **참고:** `touchstart`/`touchmove`/`touchend`는 `input.js`에서 이미 처리 중(좌표 매핑 포함).

## 4. 난이도 게임 중 변경 시 처리 (미정)
- **현재:** 툴바 난이도 토글은 `Game.saveData.difficulty`만 갱신. 실제 적용은 전투 시작(`BattleScene.init`)에 1회 — 진행 중인 매치엔 반영 안 되고 **다음 전투/컨티뉴부터** 적용됨.
- **문제:** 전투 중 토글한 플레이어는 즉시 효과를 기대하나 이번 판엔 변화 없음 → "버튼 고장?" 혼란 가능.
- **방향(미결정):** 전투 중에는 토글 **버튼 비활성화**(현재 처리 불가 상태를 상태로 표현). 사용자 선호. 트레이드오프: "다음 판 난이도 미리 변경"이 막힘 → "전투 중 비활성 + 전투 종료 시 자동 활성"이 무난. (대안: 즉시 재계산 — 싱글플레이라 악용 문제 없음.)
- **구현:** `Game.changeScene`에서 현재 씬이 `BattleScene`이면 `#difficulty-btn` `disabled` 토글 + 비활성 스타일.

## 5. 게임 페이싱 / 타이머 (대부분 해결)
- **해결(2026-06-13):** `updateLogic`의 `this.timer += dt` 중복 제거 → timer가 틱당 1씩 정직하게 증가. 관련 비교 상수 절반화로 기존 체감 보존. 리치 자동 타패는 의도적으로 ~0.85s로 조정.
- **남은 검토(선택):** 다른 페이싱 상수(`CPU_THINK_TIME`, INIT/데미지 대기 등)가 체감상 적절한지는 별도 재평가 가치 있음. 현재는 "버그 수정 전과 동일 체감" 유지가 목표였음.

## 6. 순일색/초일색 보라색 처리 (해결 — 규칙 정합성)
- **해결(2026-06-13):** `YakuLogic.isSunIlSaek`/`isChoIlSaek`에서 `color === 'purple'` 제외(빨/파/노 단색만 인정). 회귀 테스트(C-1 `SUN_IL_SAEK/CHO_IL_SAEK: 보라 단색…`) 추가.
- **실효는 사실상 없음(규칙 정합성 보강):** 색을 안 가려 12장 전부 보라(mayu_purple)인 손도 순/초일색으로 *매칭 후보*엔 올랐던 잠재 버그. 단 (1) 보라는 1종 × `TILE_COUNT_PER_TYPE: 9`장이라 12장을 못 채워 **도달 불가**, (2) 설령 만들어도 12장 동일패는 `IP_E_DAM`(입에 담을 수도 없는 엄청난 기술, 12800)이 순일색(6400)보다 높아 그게 최상위로 표시됨 → 원시 키 노출도 실제론 안 일어남.
- **주의(과거 오기 정정):** `IL_SAEK`(일색)은 "한 색 3세트 + 보라 눈썹개 3장"인 *섞인* 손이지 순수 보라가 아님 → 보라 단색은 일색도 아니다. 같은 버그 클래스: 자유 박애 평등 보라 제외, 순일색 meta.color 누락(둘 다 동일자 수정).
