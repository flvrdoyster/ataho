# 술창고 청소 (sweep) — 작업 계획

취호전의 술창고 청소 미니게임 재현. 아타호가 술집주인의 지시로 창고 바닥의 먼지를 쓸어내며
외상 술값을 갚는다. (레퍼런스: 취호전 원작 영상)

## 게임 규칙 (레퍼런스 분석)

- 격자형 방바닥에 먼지 낀 타일이 깔려 있고, **밟고 지나간 칸이 깨끗해진다**
- **빗자루질은 한 칸 이동할 때마다 무조건 1회** (먼지 없는 칸을 밟아도 이동해 왔으면 1회) + 효과음 재생
- 술통·상자·선반 등 장애물이 동선을 막는다
- 방의 먼지를 전부 지우면 클리어 → 문으로 나가 다음 방으로 (여러 방 연속)
- 방 클리어 시 보수(번 돈) 지급 — **걸음 수가 최단 루트에 가까울수록 보수가 높다**
- 클리어 후 술집주인/아타호의 평가 대사

## 기술 베이스

- 엔진: `world/engine.js` (타일 렌더·카메라·충돌·트리거·말풍선 UI·게임 루프)
- 캐릭터: `sweep/char_sweep.js` — 술창고 전용 컨트롤러 (idle → moving → sweeping 상태 기계)
  - `window.GRID_CELL_TILES = 3` → **한 칸 = 48px (3×3 맵타일 슬래브)**
  - 근거: 새로 찍은 맵의 바닥 판석이 3×3 타일 반복 블록, 캐릭터 스프라이트 폭(48px)과 일치
  - `window.GRID_ORIGIN = {x:2, y:2}` — 플레이필드 좌상단 슬래브. startPos가 격자에서 어긋나면
    컨트롤러가 가장 가까운 빈 칸으로 자동 스냅
  - 범용 `world/char/char_grid.js`에도 `GRID_CELL_TILES` 옵션 추가해둠 (기본 1 = 기존 동작)
- 맵: `world/maps/souko/` — 베이스 맵 1장 (36×21 타일, 플레이필드 **11×6칸**, gx2~34 / gy2~19)
  + 스테이지별 장애물 조합(방해물 스프라이트)만 교체
- 스프라이트: `world/char/ataho-walk.png`(이동), `world/char/ataho-sweep.png`(빗자루질 48×64 3프레임, 정면)
  - 빗자루질 모션은 **매 이동 후 1회 재생** + 효과음 (`sweep/sweep.mp3`)
  - 스프라이트는 판석 하단보다 한 타일(16px) 위에 그려짐 (`SPRITE_OFFSET_Y: -16`)
- 진입점: `sweep/index.html` (파일 직접 열기로 동작, `MAP_BASE='../world/'`)

### 레퍼런스 실측 타이밍 (2026-07-10, 영상 60fps 프레임 차분 분석)

- 원작 게임 틱: **20Hz** (60fps 영상에서 3프레임마다 화면 변화)
- 한 칸 이동: 24px(원본 해상도)를 8px×3틱 = **약 0.18초**
- 빗자루질: 이동 직후 제자리 4틱 = **약 0.27초** (스윙 3프레임 + 복귀)
- 키 홀드 시 스텝 주기: **한 칸당 약 0.45초** (26~27프레임 간격으로 일정)
- `char_sweep.js`의 `MOVE_DURATION: 0.18` / `SWEEP_DURATION: 0.27`로 반영
